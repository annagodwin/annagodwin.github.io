Is My PDF Really a PDF? Do I Need OCR?

  • November 7, 2022

Lately, I’ve been in what I like to call “PDF Land”.

One PDF extraction task turned into another and now I have a set of tools and code-snippets (pymupdf, tabula-py, etc…) that I like to use to extract info from PDFs and an interest in the field of Visual Document Understanding (VDU). I’ll cover these topics and tools in future posts, but…

First thing’s, first. Can I even use these PDF extraction tools out of the box?

A lot of PDF extraction tools and libraries require for the PDF to be, well, a PDF1. What do you mean – this file has a .pdf extension, surely it’s a PDF! Unfortunately, even though the file is saved as a PDF, it doesn’t mean it always has the properties of a PDF file under the hood that allow for these nice, useful tools to be used.

So what falls into this category (saved as a PDF without those nice PDF properties)? Scanned images saved as PDF files.

Thankfully there’s OCR (Optical Character Recognition) technology that allows for you handle these cases. But in this quick post, let’s talk about identifying if a file needs OCR.

You can do this quickly without needing to download any software or libraries other than a PDF reader:

  1. Open the file in a PDF reader.

  2. Call Cmd+F (Ctrl+F) and type in a word you clearly see on the page.

    a) If your PDF reader highlights the word as expected, you’re in business and can use these PDF extraction tools. (This means the embedded PDF properties are inside the PDF file).

    b) If you PDF doesn’t highlight the word as expected, you will need to use an OCR method before you proceed with PDF extraction tools.
